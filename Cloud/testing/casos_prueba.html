Diseño de Casos de Prueba

Testing de Caja Blanca para Tipos de Datos y Estructuras en Python

Principios Fundamentales

Partición de Equivalencia

No podemos probar *todos* los números. Agrupamos los inputs en "clases" que deberían tener el mismo comportamiento.

Ejemplo (Edad):

  • Clase 1: Negativos (Error)
  • Clase 2: 0-17 (Menor de edad)
  • Clase 3: 18+ (Mayor de edad)

Solo necesitamos probar un valor de cada clase (ej. -5, 10, 25).

Análisis de Valores Límite (BVA)

Los bugs se esconden en los bordes. Esta técnica complementa la anterior probando los *límites* de cada clase.

Ejemplo (Edad):

  • Límites de 0-17: -1, 0, 1, 16, 17
  • Límites de 18+: 17, 18, 19

Combinando, los valores clave a probar son: -1, 0, 17, 18.

Testing de Números (int, float)

def obtener_descuento(edad: int):
    if edad < 18:
        return 0.50  # 50% (Joven)
    if edad >= 65:
        return 0.75  # 75% (Senior)
    return 0.10      # 10% (Adulto)

Clases de Equivalencia

  • Edad < 18
  • 18 <= Edad < 65
  • Edad >= 65
  • (Inválida) Edad < 0

Casos de Prueba (BVA)

  • Negativos (Inválido):
    • -1 (Límite)
    • -10 (Representante)
  • Límite Joven/Adulto:
    • 17 (Resultado: 0.50) (Límite)
    • 18 (Resultado: 0.10) (Límite)
  • Límite Adulto/Senior:
    • 64 (Resultado: 0.10) (Límite)
    • 65 (Resultado: 0.75) (Límite)
  • Robustez (Tipos):
    • "veinte" (TypeError)
    • None (TypeError)

Testing de Cadenas (str)

def validar_password(password: str):
    if len(password) < 8:
        return False
    if "*" in password:
        return False
    return True

Clases de Equivalencia

  • Longitud < 8
  • Longitud >= 8
  • Contiene "*"
  • No contiene "*"

Casos de Prueba (BVA)

  • Límite de Longitud:
    • "1234567" (len=7, False) (Límite)
    • "12345678" (len=8, True) (Límite)
  • Límite Vacío:
    • "" (len=0, False) (Límite)
  • Contenido Inválido:
    • "password*" (Falso)
    • "*" (Falso)
  • Válido:
    • "password_valido_123" (True)
  • Robustez (Tipos):
    • None (TypeError)
    • 12345678 (int)

Testing de Colecciones (list, dict)

def calcular_promedio(notas: list[float]):
    if not notas:
        return 0.0
    return sum(notas) / len(notas)

Clases de Equivalencia

  • Lista vacía
  • Lista con un elemento
  • Lista con N elementos

Casos de Prueba (Límites de Cardinalidad)

  • Cero (Límite):
    • [] (Resultado: 0.0) (Límite)
  • Uno (Límite):
    • [10.0] (Resultado: 10.0) (Límite)
  • N (Típico):
    • [5.0, 7.0, 9.0] (Resultado: 7.0)
  • Robustez (Contenido):
    • ["hola", "mundo"] (TypeError)
    • [10, None, 5] (TypeError)
  • Robustez (Input):
    • None (AttributeError por len)

Testing de Estructuras: Bucles (for, while)

def sumar_positivos(numeros: list[int]):
    total = 0
    for num in numeros:
        if num > 0:
            total += num
    return total

Cobertura de Caminos del Bucle

La clave es probar la cardinalidad del bucle. Esto se conoce como "Basis Path Testing" (Pruebas de Caminos Base).

Casos de Prueba (0-1-N)

  • Cero (0) iteraciones:
    • Probar que el bucle no se ejecuta.
    • Input: []
    • Resultado esperado: 0
  • Una (1) iteración:
    • Probar que el bucle se ejecuta exactamente una vez.
    • Input: [5] (Resultado: 5)
    • Input: [-3] (Resultado: 0)
  • N iteraciones (Típico):
    • Probar que el bucle se ejecuta múltiples veces.
    • Input: [10, -5, 3] (Resultado: 13)

Testing de Estructuras: Condicionales (if, and, or)

def puede_entrar(edad: int, es_socio: bool):
    # "Short-circuiting" (cortocircuito)
    if edad >= 18 or es_socio:
        return True
    return False

Cobertura de Decisión Modificada (MCDC)

Necesitamos probar que *cada sub-condición* (edad >= 18, es_socio) puede afectar independientemente el resultado.

Casos de Prueba (MCDC)

Buscamos los casos mínimos para probar todas las ramas.

  • Caso 1 (T, F): edad = 20, es_socio = False
    • (T or F) -> Resultado: True
  • Caso 2 (F, T): edad = 16, es_socio = True
    • (F or T) -> Resultado: True
  • Caso 3 (F, F): edad = 16, es_socio = False
    • (F or F) -> Resultado: False

Nota: (T, T) no es estrictamente necesario, ya que el cortocircuito del or hace que el segundo True sea irrelevante si el primero ya lo es.

Testing de lambda y Funciones de Orden Superior

# La 'Callable' es una función o lambda
from typing import Callable

def procesar_items(
    items: list,
    filtro: Callable[[any], bool]
):
    return [item for item in items if filtro(item)]

Casos de Prueba (del Filtro)

No probamos la lambda, probamos la función procesar_items dándole diferentes lambdas que simulan los límites:

  • Filtra Todo (Límite):
    • procesar_items([1, 2, 3], lambda x: False)
    • Resultado: []
  • No Filtra Nada (Límite):
    • procesar_items([1, 2, 3], lambda x: True)
    • Resultado: [1, 2, 3]
  • Filtro Normal:
    • procesar_items([1, 5, 10], lambda x: x > 3)
    • Resultado: [5, 10]
  • Robustez (Excepción):
    • procesar_items(["a", 1], lambda x: x > 0)
    • (Debería lanzar un TypeError, y debemos probarlo con pytest.raises)

Resumen: Su Checklist de Pruebas

Por Tipo de Dato

  • Números (int, float):
    • Cero (0)
    • Positivo (ej. 10)
    • Negativo (ej. -10)
    • Valores Límite (ej. 17, 18)
    • None
  • Cadenas (str):
    • Vacía ("")
    • Un carácter (si aplica)
    • Límite de longitud (min/max)
    • Caracteres especiales / Unicode
    • None
  • Colecciones (list, dict):
    • Vacía ([], {})
    • Un elemento
    • N elementos
    • None (como input)

Por Estructura de Control

  • Bucles (for, while):
    • 0 iteraciones (no entra al bucle)
    • 1 iteración
    • N iteraciones (típico)
  • Condicionales (if, and, or):
    • Probar cada camino True y False.
    • Probar cada sub-condición (MCDC).
  • Excepciones (try/except):
    • Probar el camino try (no hay error).
    • Probar el camino except (forzar el error).
    • Probar el camino finally (en ambos casos).
